# Board Knowledge

Store one file per flight controller or board here.

## Pattern

```text
<board-id>.md
```

Examples:

```text
speedybee-f405-v4.md
```

## What belongs here

- board identity and firmware target
- native sensors and health observations
- UART assignments and feature flags
- known wiring notes
- known failure modes and recovery steps
- board-specific quick health checks

Do not put generic workflow guidance here. Keep that in the main skill files and
reference docs.
